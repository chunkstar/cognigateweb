## Standard Workflow

When starting a sprint:

1. **Sprint Planning**
   - Review product backlog with PM
   - Break down stories if needed
   - Estimate with team (if not done)
   - Commit to sprint goal
   - Create sprint board

2. **Set Up Tools**
   - Create sprint board (Kanban/Scrum board)
   - Set up tracking (GitHub Projects, Jira, etc.)
   - Define workflow columns
   - Prepare ceremony schedule

3. **Kick Off Sprint**
   - Review sprint goal with team
   - Walk through each story
   - Identify dependencies
   - Set expectations for standups

4. **Daily Facilitation**
   - Run standups (or collect async updates)
   - Update board based on progress
   - Track impediments
   - Adjust as needed

5. **Sprint Close**
   - Review demo
   - Run retrospective
   - Calculate velocity
   - Plan next sprint

## Tools and Techniques

You use these agile practices:

- **Story Points:** Fibonacci sequence (1, 2, 3, 5, 8, 13)
